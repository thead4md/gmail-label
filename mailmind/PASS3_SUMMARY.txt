IMPLEMENTATION DETAILS
======================

1. RULES ENGINE (processing/rules.py)

   Structure:
   - Rule: dataclass with name, match_fn, labels, score_delta, confidence
   - RuleMatch: result with rule_name, matched, labels, score_delta, confidence, reason
   - RulesEngine: evaluates all rules, returns list of RuleMatch results

   Baseline Rules (5):
   - known_finance_sender: Finance domains (PayPal, Stripe, Revolut, OTP, etc.)
     → +10 score, 95% confidence
   - calendar_invite: Calendar MIME types or subject keywords
     → +5 score, 85% confidence
   - newsletter_unsubscribe: Unsubscribe link signals in body
     → -15 score, 90% confidence
   - directly_addressed: Direct recipients (not list/noreply senders)
     → +10 score, 95% confidence
   - mass_cc_penalty: More than 5 recipients
     → -10 score, 92% confidence

   Extensibility:
   - Caller can register_rule() to add custom rules at runtime
   - All rules are deterministic (no network/ML)
   - Easy to add new heuristics


2. PRIORITY SCORER (processing/scorer.py)

   Scoring Formula:
     score = base_score + rule_contribution + recency_bonus + sender_trust + penalties
     (clamped to 0-100)

   Label Base Scores:
     URGENT: 80, WORK: 60, FINANCE: 55, PERSONAL: 50,
     NOTIFICATION: 30, DEFER: 20, NEWSLETTER: 10, SPAMCANDIDATE: 5,
     DEFAULT: 30

   Adjustments:
     - Rule contribution: sum of (rule.score_delta * rule.confidence) for matched rules
     - Recency bonus: +5 if within 24h, decay to 0 by 48h
     - Sender trust: 0-10 boost if sender_reputation available
     - Newsletter penalty: -20 if marked as newsletter
     - Mass email penalty: -5 if mass CC

   Output:
     ScoreResult with:
       - total_score (0-100)
       - breakdown_text (human-readable)
       - to_dict() for JSON serialization

   Debug:
     - Full breakdown printed to console/logs
     - All intermediate calculations visible


3. SAFETY POLICY (actions/safety.py)

   Guarantees:
     ✓ Never auto-delete (hardcoded)
     ✓ Never auto-archive URGENT/FINANCE/PERSONAL
     ✓ Respect protected senders/domains
     ✓ Strict dry-run mode (no Gmail API calls)
     ✓ Rate limiting (default 50 actions/hour)

   Decision Output:
     SafetyDecision with:
       - action (name)
       - allowed (bool)
       - reason (explanation)
       - logs (audit trail)

   Dry-Run Mode:
     - SafetyPolicy(dry_run=True) logs decisions but never executes
     - Perfect for testing and safe defaults


4. ACTION EXECUTOR (actions/executor.py)

   Confidence Thresholds:
     - label: 0.65
     - star: 0.75
     - mark_important: 0.75
     - archive: 0.85
     - delete: 1.00 (never, by design)

   Actions Supported:
     - apply_label(label_name) — create if needed
     - star_message()
     - mark_important()
     - archive_message() — removes INBOX label

   Safety Integration:
     1. Check SafetyPolicy.evaluate()
     2. Verify confidence threshold met
     3. If dry-run, log decision
     4. Else: execute Gmail API call with retry/backoff
     5. Log action (attempted/executed/failed)

   Rate Limiting:
     - In-memory tracking of action timestamps
     - Respects SafetyPolicy.max_actions_per_hour

   Gmail API Isolation:
     - All Gmail API calls in executor module only
     - Labeling, starring, archiving use message.modify()


5. PIPELINE ORCHESTRATOR (processing/pipeline.py)

   Flow:
     Email
       ↓
     RulesEngine.evaluate()         → List[RuleMatch]
       ↓
     PriorityScorer.compute_score() → ScoreResult
       ↓
     Determine primary label
       ↓
     Create Prediction model
       ↓
     Database.save_prediction()
       ↓
     [Optional] ActionExecutor.suggest_action()
       ↓
     [Optional] ActionExecutor.execute_action()
       ↓
     Return Prediction (persisted)

   MVP Defaults:
     - model = "rules"
     - confidence = 0.85
     - ml_confidence = None (TODO: Phase 4)
     - llm_confidence = None (TODO: Phase 5+)

   Extension Points (stubbed for future):
     - add_ml_stage() — Phase 4 ML integration
     - add_llm_stage() — Phase 5+ LLM integration
     - add_feedback_loop() — Feedback processor


6. DEBUG HARNESS (scripts/debug_pass3_pipeline.py)
